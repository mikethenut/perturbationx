highlight_leading_nodes(dataset: str, cutoff=0.8, attr='contribution', abs_value=True, include_shortest_paths='none', path_length_tolerance=0, include_neighbors=0, neighborhood_type='union')

Highlight leading nodes.

Parameters:

  • dataset (str) – The dataset to highlight leading nodes for.

  • cutoff (float, optional) – The cutoff to use when determining leading nodes. Defaults to 0.8.

  • attr (str, optional) – The attribute to use when determining leading nodes. Defaults to “contribution”.

  • abs_value (bool, optional) – Whether to use the absolute value of the attribute. Defaults to True.

  • include_shortest_paths (str, optional) – If “directed”, the directed shortest paths between the nodes will be included. If “undirected”, the undirected shortest paths between the nodes will be included. If “none”, no shortest paths will be included. Defaults to “none”.

  • path_length_tolerance (int | float, optional) – The tolerance for the length of the shortest paths. If 0, only the shortest paths are returned. If length_tolerance is an integer, it is interpreted as an absolute length. If length_tolerance is a float, it is interpreted as a percentage of the length of the shortest path. Defaults to 0.

  • include_neighbors (int, optional) – The maximum distance from leading nodes that neighbors can be. If 0, no neighbors will be included. If 1, only the neighbors of the nodes will be included. If 2, the neighbors of the neighbors of the nodes will be included, and so on. Defaults to 0.

  • neighborhood_type (str, optional) – The type of neighborhood to include. Can be one of “union” or “intersection”. If “union”, all nodes within the maximum distance from any leading node are returned. If “intersection”, only nodes within the maximum distance from all leading nodes are returned. Defaults to “union”.

Raises:

  • ValueError – If include_shortest_paths is not “directed”, “undirected”, or “none”. If max_distance is less than 0 or neighborhood_type is not “union” or “intersection”. If length_tolerance is not a number or is negative.

  • CyError – If a CyREST error occurs.

Returns:

SUID of the network.

Return type:

int

Extract leading nodes.

Parameters:

  • dataset – The dataset to extract leading nodes for. :type dataset: str

  • cutoff (float, optional) – The cutoff to use when determining leading nodes. Defaults to 0.8.

  • attr (str, optional) – The attribute to use when determining leading nodes. Defaults to “contribution”.

  • abs_value (bool, optional) – Whether to use the absolute value of the attribute. Defaults to True.

  • inplace (bool, optional) – Whether to extract the leading nodes in-place. Defaults to True. If True, the network will be modified by hiding all nodes and edges that are not leading nodes. If False, a new network will be created with only the leading nodes.

  • include_shortest_paths (str, optional) – If “directed”, the directed shortest paths between the nodes will be included. If “undirected”, the undirected shortest paths between the nodes will be included. If “none”, no shortest paths will be included. Defaults to “none”.

  • path_length_tolerance (int | float, optional) – The tolerance for the length of the shortest paths. If 0, only the shortest paths are returned. If length_tolerance is an integer, it is interpreted as an absolute length. If length_tolerance is a float, it is interpreted as a percentage of the length of the shortest path. Defaults to 0.

  • include_neighbors (int, optional) – The maximum distance from leading nodes that neighbors can be. If 0, no neighbors will be included. If 1, only the neighbors of the nodes will be included. If 2, the neighbors of the neighbors of the nodes will be included, and so on. Defaults to 0.

  • neighborhood_type (str, optional) – The type of neighborhood to include. Can be one of “union” or “intersection”. If “union”, all nodes within the maximum distance from any leading node are returned. If “intersection”, only nodes within the maximum distance from all leading nodes are returned. Defaults to “union”.

Raises:

  • ValueError – If include_shortest_paths is not “directed”, “undirected”, or “none”. If max_distance is less than 0 or neighborhood_type is not “union” or “intersection”. If length_tolerance is not a number or is negative.

  • CyError – If a CyREST error occurs.

Returns:

SUID of the network.
