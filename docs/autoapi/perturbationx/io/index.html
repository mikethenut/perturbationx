class perturbationx.io.RelationTranslator(mappings=None, allow_numeric=True)

Class for translating relations to numeric values. By default, relations “1”, “increases”, “->”, “directlyIncreases”, and “=>” are mapped to 1.0, while relations “-1”, “decreases”, “-|”, “directlyDecreases”, and “=|” are mapped to -1.0. Relations that cannot be mapped to a numeric value will be parsed as 0.0.

Parameters:

  • mappings (dict, optional) – Dictionary of additional relation to numeric value mappings. It extends and overrides the default mappings.

  • allow_numeric (bool, optional) – If True, relations will be parsed as numeric values if they cannot be found in the mappings dictionary. Defaults to True.

perturbationx.io.write_dsv(graph: networkx.DiGraph, filepath: str, edge_type='all', delimiter='\t', data_cols=DEFAULT_DATA_COLS, header=None)

Write a networkx.DiGraph to a delimited file.

Parameters:

  • graph (nx.DiGraph) – The graph to write.

  • filepath (str) – The path to write the file to.

  • edge_type (list | str, optional) – List of edge types to include. If “all”, all edges will be included. Defaults to “all”.

  • delimiter (str, optional) – The delimiter to use in the DSV file. Defaults to ” “.

  • data_cols (list, optional) – List of data columns to include. Columns not in [“subject”, “object”, “relation”, “type”] will be ignored. Defaults to [“subject”, “object”, “relation”, “type”].

  • header (list, optional) – List of header values to use. Must be given in the same order as data_cols. Defaults to None.

Raises:

ValueError – If the length of the header list does not match the length of the data_cols list.
